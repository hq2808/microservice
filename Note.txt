AuthService → PostgreSQL (MongoDB không phù hợp, vì người dùng và xác thực cần quan hệ chặt chẽ, transaction)

CustomerService → MySQL (Nếu dữ liệu khách hàng có schema ổn định → MySQL tốt)

ProductService → MongoDB (Nếu sản phẩm có nhiều thuộc tính thay đổi theo loại (size, màu, spec, feature…))

OrderService → MySQL (Order có quan hệ nhiều bảng, transaction quan trọng → MySQL/PostgreSQL)

PaymentService → PostgreSQL (Transaction tài chính → cần ACID, nên không dùng MongoDB)
